# 发票抵扣工具集 — 使用说明

面向财务场景的本地小工具，用于在 Excel 中**从摘要提取数电发票号码**、**与税额表核对**，以及**生成进项税抵扣勾选导入模板**。主程序为 `tool.py`，图形界面基于 Windows（含拖拽、另存为、自动打开文件）。

---

## 一、运行环境

| 项目 | 说明 |
|------|------|
| 操作系统 | **Windows**（使用 `os.startfile` 打开 Excel 结果） |
| Python | 建议 **3.10+**（开发验证为 3.13） |
| 界面 | 自带 **tkinter**，无需单独装 Qt |

### 1.1 安装依赖

在项目目录下执行：

```bash
py -3 -m pip install -r requirements.txt
```

依赖包括：`openpyxl`、`pandas`、`tkinterdnd2`、`xlrd`（**读取 `.xls` 必需**）。

> 若系统里 `python` 指向 2.7，请用 **`py -3`** 或 IDE 中选 **Python 3** 解释器。

### 1.2 启动程序

```bash
py -3 tool.py
```

## 二、界面说明

- **A 区 / B 区 / C 区**：将 Excel **分别拖入**对应区域，**每次建议只拖一个文件**；若一次拖入多个，程序会使用**第一个** Excel，并在日志中提示。
- **按钮 1 / 按钮 2**：开始执行对应功能。若某区未拖入文件，处理时会弹出**系统文件选择**对话框补选。
- **运行日志**：处理过程与错误信息会显示在下方文本框；出错时仍会弹出错误提示，**不会强制长时间等待**，可修正后再次操作。

## 三、功能一：发票提取 + 税额核对

### 3.1 需要什么文件

| 区域 | 含义 | 说明 |
|------|------|------|
| **A 区** | 原始账务表（待处理表） | 工作表中需能自动识别到表头里的 **「摘要」** 列；程序在摘要旁插入「提取发票号」，并从摘要文本中用规则抽取长数字串作为发票号。 |
| **B 区** | 抵扣发票信息表（税额对照） | 需为 **Excel 首行表头** 中包含列名 **`数电发票号码`**、**`有效抵扣税额*`**（与导出/税务系统字段一致），用于按发票号匹配税额。 |

支持 **`.xlsx`** 与 **`.xls`**。

### 3.2 程序会做什么（概要）

1. 在 A 表中定位「摘要」列，写入/拆分「提取发票号」，并对明显异常（如长度非 20 位、重复）做**颜色与批注**标记。  
2. 用 B 表按「数电发票号码」写入税额匹配；未匹配单元格会标出并带批注提示核对票种、是否已抵扣等。  
3. 按既定版式整理边框、列、按凭证汇总**差额**等（与当前单位表样绑定，表结构大幅改动可能导致结果不符合预期）。  
4. 弹出 **Windows「另存为」** 对话框；默认建议文件名为 **`发票提取 + 税额核对完成表.xlsx`**。保存成功后**自动打开**该文件。  
5. 若在另存为时**取消**，则**不会**把结果写入磁盘。

## 四、功能二/三：生成抵扣勾选模板

### 4.1 需要什么文件

| 区域 | 含义 |
|------|------|
| **A 区** | 原始账务表（功能2） 或 自定义发票表（功能3）。 |
| **B 区** | 抵扣勾选增值税发票信息表 |
| **C 区** | 进项税发票抵扣勾选**导入模版**（空模板或官方模版） |

同样支持 `.xlsx` / `.xls`。

### 4.2 A 表约定（功能二：基于原始账务表）

程序会按以下优先级自动从 A 表提取发票号：

**优先级 1：已用功能一处理过的表（推荐）**  
如果表头包含 **「提取发票号」** 列，程序会直接从该列的下一行开始读取所有的发票号。这样可以保证功能一提取出的发票全部纳入，**完美解决因拆分行导致的发票遗漏问题**。

**优先级 2：未用功能一处理的表（方案一）**  
如果未找到「提取发票号」列，但找到 **「摘要」** 表头，则会：  
1. 左邻列 + 摘要列文本组合与上一行完全相同时跳过去重。  
2. 取出摘要中所有长度 ≥ 15 的数字段；**同一行若有多段，全部进入列表**。

**优先级 3：兼容模式**  
若上述两者均未找到，则尝试从 **D 列**、**第 9 行** 起读取（每行取第一段 15～30 位数字），与旧版兼容。

### 4.3 A 表约定（功能三：基于自定义发票表）

如果你有自己整理的发票表格，想直接通过它来生成抵扣模板，请使用**功能3**：
1. **强制从 A 列（第 1 列）提取**。
2. 数据从 **第 2 行** 开始。
3. 单元格内可以是纯发票号，也可以是包含发票号的文本（程序会自动提取 15～30 位的数字段）。

### 4.4 B 表列位置约定（从第 2 行数据起）

程序按 **固定列号** 从 B 表活动工作表读数（第 1 行视为表头之上或结构区，**数据从第 2 行开始**）：

| 列 | 用途 |
|----|------|
| 第 2 列 | 数电发票号码 |
| 第 3 列 | 作为与 A 表提取号码**匹配的键** |
| 第 4 列及之后 | 发票代码、开票日期、金额、税额、购销方、发票类型等，按代码中字段名写入 C 模板 |

若你单位导出的 B 表列顺序与上述不一致，需先调整列或联系维护人员改程序。

### 4.5 输出

- 在 C 表结构基础上填充勾选行，默认另存为建议名：**`✅已生成_进项税抵扣模板.xlsx`**（可在对话框中改名、改路径）。  
- 保存后同样会**自动打开**。

## 五、打包为 EXE（可选）

项目内已提供 PyInstaller 配置，可在项目根目录双击 **`build_exe.bat`**，或在命令行执行：

```bash
cd /d d:\jinbu\wenyi
py -3 -m PyInstaller --noconfirm --clean InvoiceTaxTool.spec
```

生成的可执行文件一般在 **`dist\InvoiceTaxTool.exe`**。首次运行若较慢，属单文件解压正常现象；若杀毒软件拦截拖拽组件，可将程序加入信任区。

打包前请确保已安装：

```bash
py -3 -m pip install pyinstaller -r requirements.txt
```

## 六、常见问题

1. **`ModuleNotFoundError: No module named 'openpyxl'`**  
   使用 Python 3 并执行：`py -3 -m pip install -r requirements.txt`。

2. **缺少 `xlrd`，无法读 `.xls`**  
   安装：`py -3 -m pip install "xlrd>=2.0.1"`。

3. **功能一提示未找到「摘要」列**  
   仅影响**功能一**。**功能二**会优先用摘要做方案一解析；若无摘要则会尝试 **D 列**。

4. **功能一 B 表报错列名不存在**  
   确认存在列 **`数电发票号码`**、**`有效抵扣税额*`**（含星号），且为第一行表头。

5. **拖拽无反应**  
   确认安装 `tkinterdnd2`；EXE 环境下检查是否被杀软拦截；也可改用按钮 + 文件选择框。
